# Objective

Implement weighted scoring in `matching/scoring.py`. Replace the `NotImplementedError` stubs with real deterministic scoring logic. Input is `UserProfile` (not raw `AssessmentRequest`).

# Implementation Plan

1. Implement `score_resource(resource, profile) -> float` with weighted factors:
   - **Barrier alignment** (40%): resource category matches user's primary barriers
   - **Geographic proximity** (20%): haversine distance from user zip centroid
   - **Transit accessibility** (15%): penalize resources requiring Sunday/night access for transit-dependent users (M-Transit: NO Sunday, weekday ~5am-9pm only)
   - **Schedule compatibility** (15%): resource hours vs user available_hours
   - **Industry match** (10%): resource/employer industry vs target_industries
2. Implement `rank_resources(resources, profile) -> list[Resource]` -- sort descending by score
3. Add score band classification: strong_match (80-100), good_match (60-79), possible_match (40-59), weak_match (0-39)
4. All scoring functions must be pure and deterministic (no randomness, no external calls)
5. Write tests first (TDD)
