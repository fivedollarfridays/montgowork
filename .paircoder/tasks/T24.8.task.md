# Objective

Abstract the LLM streaming client so contributors can run locally with either
Anthropic or OpenAI — no forced provider. Controlled by a single `LLM_PROVIDER`
env var; keys for unused providers are optional.

# Implementation Plan

1. Add `llm_provider`, `openai_api_key`, `openai_model` to `app/core/config.py`
2. Create `app/barrier_intel/llm_client.py` — factory returning a provider-agnostic async stream generator
3. Update `stream.py` to call `llm_client` instead of `AsyncAnthropic` directly
4. Update `backend/.env.example` with all three provider env vars + instructions
5. Tests: provider selection logic, fallback when key missing

# Acceptance Criteria

- [ ] `LLM_PROVIDER=anthropic` (default) uses Anthropic Claude as before
- [ ] `LLM_PROVIDER=openai` uses OpenAI with `OPENAI_API_KEY` + `OPENAI_MODEL`
- [ ] Missing API key for selected provider raises clear startup error
- [ ] `backend/.env.example` documents all provider options
- [ ] `stream.py` unchanged interface (only internal provider swap)
- [ ] Arch check passes on all modified files
- [ ] Unit tests cover provider factory and config validation

# Verification

Run with each provider locally, confirm SSE stream works end-to-end.
