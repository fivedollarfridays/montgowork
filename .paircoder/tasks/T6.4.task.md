# Objective

Feed credit assessment results into the plan page. Show which jobs the user is eligible for now vs after credit repair. Display CreditResults component on plan page. Update ComparisonView with credit-specific data.

# Implementation Plan

- Modify src/app/plan/page.tsx:
  - Fetch credit results (from session storage or new API call if session has credit data)
  - Pass CreditAssessmentResult to CreditResults component
  - Show CreditResults section between BarrierCards and JobMatches (only if credit assessment exists)
- Modify src/components/plan/ComparisonView.tsx:
  - Use credit thresholds to show "Today: Score X" vs "In 3 Months: Score Y (estimated)"
  - Show eligible product count in "now" vs "after repair" columns
- Modify src/components/plan/JobMatchCard.tsx:
  - If credit_check_required and credit results available, show estimated days to eligibility
  - Badge: "Eligible Now" (green) vs "Eligible in X days" (amber) vs "Credit Check Required" (red)
- Use plan.eligible_now and plan.eligible_after_repair arrays to split job display
- Add "Qualified Now" and "After Credit Repair" sections to job matches area

# Target Files

- frontend/src/app/plan/page.tsx (modify)
- frontend/src/components/plan/ComparisonView.tsx (modify)
- frontend/src/components/plan/JobMatchCard.tsx (modify)
