# Objective

Build the RAG knowledge corpus and FAISS vector store powering the Barrier Intelligence
Assistant. Documents come from: (1) resources DB rows, (2) barrier playbooks (`barriers.playbook`),
(3) optional Montgomery policy notes. Implement an idempotent ingestion pipeline using
`sentence-transformers/all-MiniLM-L6-v2` and `faiss-cpu`.

# Document Schema

```python
class RagDocument(BaseModel):
    id: str                      # "resource_42" or "playbook_CHILDCARE_EVENING"
    doc_type: str                # "resource" | "playbook" | "policy_note"
    title: str
    text: str                    # Full text that gets embedded
    barrier_tags: list[str]      # ["CHILDCARE_EVENING", "CHILDCARE_DAY"]
    geography: str | None        # ZIP code or city
    schedule_type: str | None    # "day" | "evening" | "flexible" | None
    transit_accessible: bool
    impact_strength: float       # 0.0–1.0
```

# FAISS Index Setup

```python
# Model: sentence-transformers/all-MiniLM-L6-v2  (384-dim, local, free)
# Index: IndexFlatIP (inner product = cosine after L2 normalization)
index = faiss.IndexFlatIP(384)
```

# Idempotent Re-ingestion

On startup:
1. Check if `index.faiss` exists and is newer than last resources update
2. If stale/missing: rebuild from DB → embed → save
3. If fresh: load from disk

Expose `POST /api/barrier-intel/reindex` (admin-only, X-Admin-Key) to force rebuild.
Add `*.faiss` to `.gitignore`.

# Chunking Strategy

- Resources: no chunking (≤500 tokens each)
- Playbooks: split at paragraph boundaries if >500 tokens, 50-token overlap
