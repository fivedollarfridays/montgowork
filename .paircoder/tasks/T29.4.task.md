# Objective

Add a Benefits Eligibility section to the plan page showing which programs the user qualifies for, estimated monthly value, and actionable next steps per program. Positioned between barriers and cliff chart.

# Implementation Plan

1. **Create `BenefitsEligibility.tsx`**:
   - Section heading: "Benefits You May Qualify For"
   - Summary card: total estimated monthly value, count of eligible programs
   - Two groups:
     - **Enrolled**: Programs user already reported — show current value
     - **Additional Eligible**: Screener-detected — "You may also qualify" with confidence badge
   - Per-program row:
     - Program name (PROGRAM_LABELS), estimated monthly value
     - Confidence badge: likely (green), possible (amber), unlikely (gray)
     - Income headroom: "Eligible up to $X/month income"
     - Expand/collapse for application details: steps, docs, office contact, processing time, apply link
   - Disclaimer at bottom
   - Null check: render nothing when null
   - Accessibility: aria-labels on badges, semantic headings

2. **Wire into plan page**: After barrier cards, before BenefitsCliffChart

3. **Tests**: Null renders nothing, heading present, programs listed, badges styled, expand/collapse works, disclaimer present, accessible
