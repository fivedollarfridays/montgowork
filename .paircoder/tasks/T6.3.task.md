# Objective

Wire CreditForm as a conditional step in the assessment wizard. Only shows when user selects "Credit" barrier in Step 2. Dynamic step count adjusts accordingly.

# Implementation Plan

- Modify src/app/assess/page.tsx:
  - Track whether credit barrier is selected from BarrierForm
  - Dynamically build steps array: [Basic Info, Barriers, Credit (conditional), Review]
  - WizardShell step count adjusts (3 or 4 steps)
  - Credit step renders CreditForm component
  - Store credit form data in page state alongside barrier form data
- On submit:
  - If credit barrier selected, call postCredit() with CreditProfileRequest BEFORE postAssessment()
  - Store CreditAssessmentResult in state
  - Include credit data in assessment request (or store separately for plan page)
  - Pass credit results to plan page via query param or session storage
- Handle credit API errors gracefully (show error toast, allow skip)
- Update WizardShell step labels to include "Credit Check" when active

# Target Files

- frontend/src/app/assess/page.tsx (modify)
- frontend/src/components/wizard/WizardShell.tsx (may need dynamic step labels)

# Acceptance Criteria

- [x] Credit step appears ONLY when credit barrier is selected in Step 2
- [x] Deselecting credit barrier removes the credit step
- [x] Step indicator shows correct count (3 without credit, 4 with credit)
- [x] Credit form data is collected and sent to POST /api/credit/assess
- [x] Credit API errors show user-friendly message, allow continuing without credit results
- [x] Review step shows credit summary when credit data is present
- [x] Navigation (prev/next) works correctly with dynamic steps
- [x] Build and lint clean
