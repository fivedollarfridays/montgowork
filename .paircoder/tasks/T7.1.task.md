# Objective

Create an async httpx wrapper for the BrightData Datasets API v3 with typed request/response models, error handling for missing config, and full test coverage.

# Files to Update

| File | Change |
|------|--------|
| `backend/app/integrations/brightdata/client.py` | NEW: BrightDataClient class |
| `backend/app/integrations/brightdata/types.py` | MODIFY: add error classes, request/response models |
| `backend/tests/test_brightdata_client.py` | NEW: full test coverage |

# Implementation Plan

1. Write tests first: success, missing API key, HTTP errors for trigger and status
2. Add `BrightDataConfigError` and `BrightDataAPIError` to types.py
3. Implement `BrightDataClient(api_key, dataset_id)` with httpx.AsyncClient
4. `trigger_crawl(urls) -> str` — POST /datasets/v3/trigger, returns snapshot_id
5. `get_snapshot_status(snapshot_id) -> CrawlProgress | CrawlResult` — 202=running, 200=done
6. Config guard: raise BrightDataConfigError when API key empty

# Acceptance Criteria

- [x] trigger_crawl returns snapshot_id on success
- [x] get_snapshot_status differentiates 202 (running) vs 200 (ready with data)
- [x] Raises BrightDataConfigError when API key empty
- [x] Raises BrightDataAPIError on non-2xx responses
- [x] All HTTP calls mockable, 100% test coverage
- [x] Module under 400 lines, under 15 functions
