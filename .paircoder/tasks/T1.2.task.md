# Objective

Rewrite `database.py` to use raw DDL (no ORM models) for table creation and add a `seed_database()` function that loads Montgomery JSON data on first run. Wire into FastAPI lifespan.

# Files to Update

| File | Change |
|------|--------|
| `backend/app/core/database.py` | Remove Base/DeclarativeBase, add DDL_SQL, ALLOWED_COLUMNS, JSON_FIELDS, init_db(engine), seed_database(engine) |
| `backend/app/main.py` | Update lifespan to call `init_db(engine)` instead of `Base.metadata.create_all()` |

# Implementation Plan

1. Read existing `database.py` — preserve engine/session factory code
2. Remove `Base`, `DeclarativeBase`, and `init_db()` that uses `create_all()`
3. Add `DDL_SQL` string with all 6 CREATE TABLE statements
4. Add `ALLOWED_COLUMNS` dict for seed injection prevention
5. Add `JSON_FIELDS` set for list/dict serialization
6. Add `DATA_DIR` path: `Path(__file__).resolve().parent.parent.parent.parent / "data"`
7. Write `init_db(engine)` — executes DDL then calls `seed_database()`
8. Write `seed_database(engine)` — loads JSON files, filters columns, serializes JSON fields
9. Update `main.py` lifespan to use new `init_db(engine)`
10. Verify tables created and seed data loaded on startup

# Acceptance Criteria

- [x] `Base` and `DeclarativeBase` removed from database.py
- [x] 6 tables created: employers, transit_routes, transit_stops, resources, job_listings, sessions
- [x] `ALLOWED_COLUMNS` prevents arbitrary column names from JSON
- [x] `JSON_FIELDS` serializes list/dict fields with `json.dumps()` before insertion
- [x] `DATA_DIR` resolves to project-root `data/` (4 parents from database.py)
- [x] `seed_database()` is idempotent (checks `SELECT COUNT(*) FROM resources`)
- [x] FastAPI lifespan calls `init_db(engine)` on startup
- [x] App starts without errors and tables exist in SQLite
- [x] database.py stays under 300 lines
