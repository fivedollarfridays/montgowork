# Objective

Build CreditResults.tsx — displays credit assessment results including dispute pathway steps, credit repair timeline, severity indicators, and product eligibility. Shown on the plan page when credit barrier is active.

# Implementation Plan

- Create src/components/plan/CreditResults.tsx
- Accept CreditAssessmentResult as prop
- Display sections:
  1. **Credit Health Summary**: barrier_severity badge (high/medium/low), readiness.score, readiness.fico_score, readiness.score_band
  2. **Key Factors**: readiness.factors breakdown — payment_history, utilization, account_age, credit_mix, new_credit with progress bars
  3. **Dispute Pathway**: dispute_pathway.steps as numbered timeline cards — step_number, action (bold), description, with total_estimated_days header
  4. **Credit Thresholds**: thresholds as progress cards — threshold_name, current vs threshold_score, estimated_days, already_met badge (green check), confidence indicator
  5. **Product Eligibility**: eligibility list — product_name, category, required_score, status (eligible/not_eligible), gap_points, estimated_days_to_eligible
  6. **Disclaimer**: disclaimer text in muted card
- Use severity color scheme: high=red, medium=amber, low=green (from constants.ts)
- Collapsible sections for dispute pathway and eligibility (show summary, expand for details)
- Mobile responsive: stack columns on small screens

# Target Files

- frontend/src/components/plan/CreditResults.tsx (new)
- frontend/src/lib/types.ts (ensure CreditAssessmentResult type is complete)
