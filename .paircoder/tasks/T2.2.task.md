# Objective

Fix two error handling gaps in the credit proxy route:
1. `resp.json()` can raise `JSONDecodeError` on non-JSON error responses (e.g., HTML 502 from reverse proxy)
2. Only `httpx.ConnectError` is caught — `TimeoutException` and other network errors produce unhandled 500s

# Files to Update

| File | Change |
|------|--------|
| `backend/app/routes/credit.py` | Catch broader httpx errors, wrap resp.json() in try/except |

# Implementation Plan

1. Wrap `resp.json().get("detail", ...)` in try/except for JSONDecodeError
2. Add `httpx.TimeoutException` as a separate except clause with a clear "timeout" message
3. Catch remaining `httpx.HTTPError` (base class) as fallback with generic network error message
4. Write tests for each error scenario

# Acceptance Criteria

- [x] Non-JSON error response returns structured error (not 500 traceback)
- [x] `httpx.TimeoutException` returns 504 with "Credit API timed out" message
- [x] `httpx.ConnectError` still returns 503 with existing message
- [x] Other `httpx.HTTPError` subclasses return 502 with "network error" message
- [x] Test covers JSON decode error path
- [x] Test covers timeout path
- [x] arch check passes
