# Objective

Enforce the MontGoWork brand color palette (Navy/Teal/Amber/Coral/Green) across all components. Replace any default shadcn/Tailwind colors with brand tokens.

# Files to Update

| File | Action |
|------|--------|
| `frontend/tailwind.config.ts` | MODIFY: add brand color tokens |
| `frontend/src/app/globals.css` | MODIFY: update CSS custom properties |
| `frontend/src/components/**` | MODIFY: replace hardcoded colors with tokens |

# Implementation Plan

1. Define brand palette in `tailwind.config.ts`:
   - `navy`: primary dark (headings, nav) — #1B2A4A
   - `teal`: primary accent (buttons, links) — #2A9D8F
   - `amber`: warning/attention (credit barriers) — #E9C46A
   - `coral`: urgent/blockers — #E76F51
   - `green`: success/eligible — #52B788
2. Map to CSS custom properties in `globals.css`:
   - `--primary` → navy
   - `--accent` → teal
   - `--warning` → amber
   - `--destructive` → coral
   - `--success` → green
3. Audit all components for hardcoded color values:
   - Replace `text-blue-*` → `text-navy-*`
   - Replace `bg-green-*` → `bg-success`
   - Replace `text-amber-*` → `text-warning`
   - Ensure barrier cards use severity-appropriate colors
4. Verify dark mode still works (if applicable)

# Acceptance Criteria

- [x] Brand colors defined as Tailwind tokens
- [x] CSS custom properties map to brand palette
- [x] No hardcoded Tailwind default colors in components (blue-*, green-*, etc.)
- [x] Barrier cards use severity-appropriate brand colors
- [x] Buttons use teal accent color
- [x] Visual consistency across all pages
