# Caching Strategy

`cachetools.TTLCache` (in-memory, v1 — no Redis required):

```python
# Key: sha256(session_id + question + mode)[:16]
# TTL: 300s (5 min) for LLM responses
# TTL: 600s (10 min) for retrieval context
RESPONSE_CACHE: TTLCache = TTLCache(maxsize=200, ttl=300)
RETRIEVAL_CACHE: TTLCache = TTLCache(maxsize=500, ttl=600)
```

Cache hit emits `"cached": true` in context SSE event. Redis is the documented v2 upgrade path.

# Offline Evaluation Suite

Golden query schema (`golden_queries.json`):
```json
[{
  "id": "eval_001",
  "situation": {"barriers": ["CHILDCARE_EVENING", "TRANSPORTATION_NO_CAR"], "zip": "36104"},
  "question": "What should I do first to get a job?",
  "mode": "next_steps",
  "expected_properties": {
    "must_mention_barriers": ["childcare", "transportation"],
    "must_not_mention": ["legal advice", "guarantee"],
    "must_have_steps": true, "min_steps": 2, "max_steps": 4
  }
}]
```

Minimum: ≥20 queries covering all barrier combinations and both modes.

Automated checks in `run_eval.py`:
- Response mentions ≥1 expected barrier keyword
- Response contains no disallowed keywords
- Response has ≥min_steps numbered steps (`^\d+\.` regex)
- All resource names in response appear in retrieval context

Target: ≥85% pass rate.
