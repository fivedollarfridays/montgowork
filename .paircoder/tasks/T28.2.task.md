# Objective

Add eligibility-based resource filtering and badging. Given a user's BenefitsProfile (income, household size, enrolled programs), determine whether each matched resource is "likely eligible", "check eligibility", or unknown — and display badges on barrier cards.

Currently resources are matched by barrier category + affinity keywords only. This adds a second dimension: does the user actually qualify for this resource?
