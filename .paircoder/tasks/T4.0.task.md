# Objective

Implement async query helpers so scoring, filters, and routes can fetch data from SQLite. Currently `database.py` has only DDL, seeding, and validation -- zero query functions exist.

# Files to Update

| File | Change |
|------|--------|
| backend/app/core/queries.py | NEW -- all query functions |
| backend/tests/test_queries.py | NEW -- tests for every query function |

# Implementation Plan

1. Create `backend/app/core/queries.py` with async query helpers:
   - `get_all_resources(session) -> list[dict]`
   - `get_resource_by_id(session, id) -> dict | None`
   - `get_resources_by_category(session, category) -> list[dict]`
   - `get_all_transit_routes(session) -> list[dict]`
   - `get_transit_routes_by_area(session, lat, lng, radius_miles) -> list[dict]`
   - `get_all_employers(session) -> list[dict]`
   - `get_session_by_id(session, session_id) -> dict | None`
   - `create_session(session, session_data) -> str` (returns UUID)
2. All queries use `text()` with parameterized SQL (no f-strings)
3. Write tests (TDD) using the existing `test_engine` fixture pattern from conftest.py
4. Each query function returns dicts (not ORM models -- we use raw DDL)

# Acceptance Criteria

- [x] All query functions implemented with parameterized SQL
- [x] No SQL injection vectors (use `:param` binding, never f-strings for values)
- [x] Tests cover each function: happy path + edge cases (empty results, not found)
- [x] Tests use existing `test_engine` fixture from conftest.py
- [x] `pytest tests/test_queries.py -v` passes
- [x] File stays under 400 lines
