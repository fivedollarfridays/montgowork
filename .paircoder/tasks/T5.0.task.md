# Objective

Bootstrap the frontend foundation: install shadcn/ui base components, write TypeScript types mirroring all backend Pydantic models, and rewrite the API client with full type safety. Every subsequent task depends on this.

# Files to Update

| File | Change |
|------|--------|
| frontend/src/components/ui/*.tsx | Install Button, Card, Badge, Input, Checkbox, Progress, Tabs, Select, Separator via shadcn CLI |
| frontend/src/lib/types.ts | All Pydantic models as TS interfaces — zero `any` types |
| frontend/src/lib/api.ts | Typed `apiFetch<T>()`, `postAssessment()`, `getPlan()`, `generateNarrative()`, `getJobs()`, `postCredit()` |
| frontend/src/lib/providers.tsx | TanStack QueryClientProvider wrapper |
| frontend/src/app/layout.tsx | Wrap with QueryClientProvider |

# Implementation Plan

1. Install shadcn/ui components: `npx shadcn@latest add button card badge input checkbox progress tabs select separator`
2. Write TypeScript types matching backend models:
   - Enums: BarrierType, BarrierSeverity, EmploymentStatus, AvailableHours
   - Interfaces: UserProfile, Resource, JobMatch, ReEntryPlan, PlanNarrative
   - Request/Response: AssessmentRequest, AssessmentResponse, CreditProfileRequest, CreditAssessmentResult
3. Rewrite api.ts with generic `apiFetch<T>()` and typed endpoint functions
4. Create QueryClientProvider in providers.tsx
5. Wrap layout.tsx with provider
