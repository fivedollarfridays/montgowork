# Objective

Generate findhelp.org capability URLs per barrier category, allowing users to discover 602+ Montgomery programs through filtered findhelp.org searches. Display "More programs available" links on each barrier card.

MVP approach — no API partnership needed. Uses findhelp's capability URL pattern to embed filtered program searches per barrier type + user zip code.

# Files to Create/Modify

| File | Purpose |
|------|---------|
| `backend/app/modules/resources/__init__.py` | Module init |
| `backend/app/modules/resources/findhelp.py` | URL generation + category mapping |
| `backend/tests/test_findhelp.py` | Tests for URL generation |
| `frontend/src/components/plan/FindhelpLink.tsx` | "More programs" link component |
| `frontend/src/components/plan/__tests__/FindhelpLink.test.tsx` | Frontend tests |
| `frontend/src/components/plan/BarrierCardView.tsx` | Add FindhelpLink to resource section |

# Implementation Plan

## 1. Backend: Category Mapping + URL Generation (`findhelp.py`, ~50 lines)

Map 7 barrier types to findhelp.org URL categories:

| BarrierType | findhelp Category Path |
|---|---|
| CREDIT | `/money/financial-assistance` |
| TRANSPORTATION | `/transit/transportation` |
| CHILDCARE | `/care/childcare` |
| HOUSING | `/housing/housing` |
| HEALTH | `/health/health-care` |
| TRAINING | `/work/job-training` |
| CRIMINAL_RECORD | `/work/help-for-the-formerly-incarcerated` |

Function: `generate_findhelp_url(barrier_type, zip_code) -> str | None`
URL pattern: `https://www.findhelp.org/{category_path}--montgomery-al?postal={zip_code}`

## 2. Backend Tests (`test_findhelp.py`)

- Each barrier type produces correct URL
- Invalid barrier type returns None
- ZIP code is embedded correctly
- URL is properly formatted (no double slashes, etc.)
- All 7 barrier types have mappings

## 3. Frontend: FindhelpLink Component (~30 lines)

Small link component: external link icon + "Find more programs on findhelp.org"
Props: `barrierType: BarrierType`, `zipCode: string`
Opens in new tab with `rel="noopener noreferrer"`

## 4. Frontend: Wire into BarrierCardView

Add FindhelpLink below the resource list in each barrier card.
Pass barrier type and zip code from plan context.

## 5. Frontend Tests

- Component renders link with correct href
- Link opens in new tab
- All 7 barrier types generate valid URLs
- Component handles missing zip code gracefully
