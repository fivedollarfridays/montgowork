# Objective

Create a commute time estimator that calculates approximate drive and transit times using haversine distance + average speed formulas. No external API needed -- uses existing ZIP centroids and transit route data from Sprint 30.

# Files to Update

| File | Change |
|------|--------|
| `app/modules/matching/commute_estimator.py` | New: estimate_commute() with drive/transit/walk times |
| `app/modules/matching/types.py` | Add `CommuteEstimate` model (drive_min, transit_min, walk_min) |
| `app/modules/matching/transit_schedule.py` | Export walk distance for commute calculation |
| `app/modules/matching/job_matcher.py` | Wire commute estimate into ScoredJobMatch |
| `tests/test_commute_estimator.py` | Unit tests for all estimate scenarios |

# Implementation Plan

1. Create `commute_estimator.py` with:
   - `estimate_commute(user_zip, job_location, transit_info) -> CommuteEstimate`
   - Drive time: `distance_miles / 25 mph * 60` (avg city driving in Montgomery)
   - Transit time: `walk_to_stop_min + wait_min(10) + ride_min(distance / 12mph) + walk_from_stop_min`
   - Walk time: `distance_miles / 3 mph * 60` (only for jobs < 2 miles)
   - Return None for modes that don't apply (e.g., walk=None if >2 miles)
2. Add `CommuteEstimate` to types.py: `drive_min: int | None`, `transit_min: int | None`, `walk_min: int | None`
3. Add `commute_estimate` field to `ScoredJobMatch` in types
4. Wire into job_matcher or pvs_scorer to compute during scoring pass
