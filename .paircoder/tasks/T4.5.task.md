# Objective

Build the plan route from scratch (greenfield -- empty router with zero endpoints). Includes Claude API client for AI-generated plan narratives, with a template fallback when API is unavailable.

# Files to Update

| File | Change |
|------|--------|
| backend/app/routes/plan.py | Add endpoints: GET /api/plan/{session_id}, POST /api/plan/{session_id}/generate |
| backend/app/ai/client.py | NEW -- Claude API client wrapper |
| backend/app/ai/prompts.py | NEW -- prompt templates for plan generation |
| backend/tests/test_plan_route.py | NEW -- plan route tests |

# Implementation Plan

1. **Plan route endpoints:**
   - `GET /api/plan/{session_id}` -- look up session, return existing plan or 404
   - `POST /api/plan/{session_id}/generate` -- generate AI narrative for existing plan
2. **Claude API client** (`ai/client.py`):
   - Async client using httpx to call Claude API
   - Structured output parsing into PlanNarrative
   - Timeout + error handling
3. **Prompt templates** (`ai/prompts.py`):
   - System prompt with Montgomery context
   - User prompt template taking UserProfile + matched resources
   - Output format specification
4. **Template fallback:**
   - If Claude API unavailable/slow, generate structured plan from matched resources without AI
   - Use barrier cards + resources to build timeline, milestones, contact info
5. Use existing type stubs: AnalysisResult, PlanNarrative from `backend/app/ai/types.py`

# Acceptance Criteria

- [ ] GET endpoint returns plan for valid session, 404 for invalid/expired
- [ ] POST endpoint generates AI narrative and stores it
- [ ] Claude API client handles timeouts and errors gracefully
- [ ] Template fallback works when API is unavailable
- [ ] Plan includes: timeline, milestones, required documents, contact info
- [ ] Uses existing AnalysisResult and PlanNarrative types
- [ ] `pytest tests/test_plan_route.py -v` passes
- [ ] All files stay under 400 lines
