# Objective

Extend BrightData precrawl from single-domain (Indeed.com) to multi-domain job discovery. Add LinkedIn, Glassdoor, ZipRecruiter, and Snagajob as crawl targets. This replaces JSearch as the secondary job data source and consolidates all external job sourcing to BrightData.

# Files to Update

| File | Change |
|------|--------|
| `app/integrations/brightdata/precrawl.py` | Add multi-domain search configs (LinkedIn, Glassdoor, ZipRecruiter, Snagajob) |
| `app/integrations/brightdata/client.py` | Add domain-specific dataset IDs support (if needed per BrightData API) |
| `app/integrations/brightdata/cache.py` | Ensure dedup handles cross-domain duplicates (same job on Indeed + LinkedIn) |
| `app/core/config.py` | Add `brightdata_job_domains` config (comma-separated domain list) |
| `tests/test_precrawl.py` | Tests for multi-domain crawl logic |
| `tests/test_brightdata_cache.py` | Tests for cross-domain dedup |

# Implementation Plan

1. Update `precrawl.py` to define domain configs: `CRAWL_DOMAINS = [{"domain": "indeed.com", ...}, {"domain": "linkedin.com", ...}, ...]`
2. Each domain gets the same 13 Montgomery keywords (warehouse, healthcare, etc.)
3. Add configurable domain list via `Settings.brightdata_job_domains` (default: "indeed.com")
4. Update `trigger_keyword_crawl()` calls to iterate over domains
5. Ensure `cache.py` dedup catches cross-domain duplicates (same company+title from Indeed and LinkedIn)
6. Add fallback: if a domain crawl fails, log warning and continue with remaining domains

# Acceptance Criteria

- [ ] Precrawl supports configurable list of job board domains
- [ ] Default config crawls Indeed.com only (backward compatible)
- [ ] Cross-domain dedup prevents duplicate listings
- [ ] Failed domain crawl doesn't block other domains
- [ ] Tests cover multi-domain config, cross-domain dedup, partial failure
- [ ] All existing BrightData tests still pass
