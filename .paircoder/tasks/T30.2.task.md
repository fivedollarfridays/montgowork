# Objective

Display transit route details on job cards so transit-dependent users see which bus routes serve each job, walk distance from the nearest stop, schedule warnings (Sunday/night gaps), and a "Plan your trip" link to Google Maps transit directions. Populate and render the existing `TransitConnection` type that is currently defined but never used.

# Context

**Current state:**
- `TransitConnection` type defined in backend `types.py` (route_number, route_name, connects_to, schedule) — never populated
- `transit_matches: list[TransitConnection]` field on BarrierCard — always empty
- `ScoredJobMatch` has `transit_accessible: bool` and `route: Optional[str]` — partially populated
- No frontend component displays transit info on job cards
- T30.1 adds `TransitInfo` with route details, walk distance, warnings, and Google Maps URL

**After T30.1:**
- Each job will have a populated `TransitInfo` object from the schedule matcher
- Routes, walk distance, warnings, and Google Maps URL available in the backend response
- Need frontend types + component to display this data

# Implementation Plan

1. **TS Types** — Add to `types.ts`:
   ```typescript
   type TransitWarning = "sunday_gap" | "night_gap" | "long_walk" | "transfer_required";

   interface RouteFeasibility {
     route_number: number;
     route_name: string;
     nearest_stop: string;
     walk_miles: number;
     first_bus: string;
     last_bus: string;
     feasible: boolean;
   }

   interface TransitInfo {
     serving_routes: RouteFeasibility[];
     transfer_count: number;
     warnings: TransitWarning[];
     google_maps_url: string | null;
   }
   ```
   Add `transit_info?: TransitInfo | null` to `ScoredJobMatch` interface.

2. **TransitInfo component** — New `TransitInfo.tsx`:
   - Collapsible section inside job card (like BenefitsEligibility "How to apply")
   - Route badges: Bus icon + route name/number for each serving route
   - Walk distance: "0.3 mi walk from [stop name]"
   - Schedule info: "First bus: 5:15 AM | Last bus: 8:45 PM"
   - Warning badges using `STATUS_BADGE_STYLES`:
     - `sunday_gap` → warning: "No Sunday service"
     - `night_gap` → warning: "Last bus before shift ends"
     - `long_walk` → warning: "1+ mi walk from stop"
     - `transfer_required` → neutral: "Transfer at Rosa Parks"
   - "Plan your trip" external link (Google Maps directions, transit mode)
   - Accessible: aria-labels on badges, aria-hidden on icons

3. **Wire into JobMatchCard** — Add TransitInfo component to the expanded view:
   - Show only when `transit_info` exists and user is transit-dependent
   - Position after salary/benefits info, before apply link
   - Collapsed by default, expand on click

4. **Backend type update** — Add `transit_info: Optional[TransitInfo] = None` to `ScoredJobMatch` in types.py, populated by T30.1's `build_transit_info()`.
