# Objective

Generate TypeScript types from Pydantic models and wire frontend pages to real API endpoints. `lib/types.ts` is currently EMPTY.

# Files to Update

| File | Change |
|------|--------|
| frontend/lib/types.ts | Generate TypeScript types matching all Pydantic models |
| frontend/app/assess/page.tsx | Wire to POST /api/assessment |
| frontend/app/plan/page.tsx | Wire to GET /api/plan/{session_id} |
| frontend/app/credit/page.tsx | Wire to POST /api/credit/assess |

# Implementation Plan

1. **Generate TypeScript types** in `lib/types.ts`:
   - All enums: BarrierType, BarrierSeverity, EmploymentStatus, AvailableHours, CrawlStatus
   - All models: AssessmentRequest, UserProfile, Resource, JobMatch, TransitConnection, BarrierCard, ReEntryPlan, ScheduleConstraints, CreditAssessmentResult
   - NO `any` types anywhere
2. **Assessment page** (`/assess`):
   - Multi-step form: zip code -> barriers -> work history -> submit
   - POST to /api/assessment
   - Display ranked matches with match bands + explanations
3. **Plan page** (`/plan`):
   - Look up session by ID (from URL params or stored from assessment)
   - GET /api/plan/{session_id}
   - Show generated action plan with timeline
4. **Credit page** (`/credit`):
   - Display credit assessment results with next steps
   - POST to /api/credit/assess
5. Use existing TanStack Query setup + api.ts client
6. Use existing shadcn/ui components
