# Objective

Harden the backend: close DB engine on shutdown, add rate limiting to assessment endpoint, sanitize BrightData inputs, validate Claude response structure.

# Files to Update

| File | Change |
|------|--------|
| backend/app/main.py | Call `close_db()` in lifespan shutdown |
| backend/app/routes/assessment.py | Add simple in-memory rate limit (10 req/min per IP) |
| backend/app/integrations/brightdata/cache.py | Add string length truncation |
| backend/app/ai/client.py | Guard against empty `message.content` |

# Implementation Plan

1. **main.py**: Add `await close_db()` after yield in lifespan
2. **assessment.py**: Simple dict-based rate limiter — IP + timestamp, 10 req/min, return 429
3. **cache.py**: Truncate title (500), company/location (200), description (5000), URL (2000)
4. **client.py**: `if not message.content: raise ValueError("Empty response")`
