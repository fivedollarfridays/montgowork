# Objective

Prevent SQL injection vectors in `seed_database()` by validating table names and column names against known allowlists before string interpolation.

# Files to Update

| File | Change |
|------|--------|
| `backend/app/core/database.py` | Add table name validation, column name validation |

# Implementation Plan

1. In `seed_database()`, before the INSERT, assert `table in ALLOWED_COLUMNS`
2. Validate that all column names in `clean.keys()` are in the `ALLOWED_COLUMNS[table]` set (already true by construction, but make it explicit)
3. Write tests for the validation (test that unknown table raises, test that disallowed column is filtered)

# Acceptance Criteria

- [x] `seed_database()` raises ValueError if table name not in ALLOWED_COLUMNS
- [x] Column names are validated against ALLOWED_COLUMNS before SQL interpolation
