# Objective

Wire the eligibility screener into `generate_plan()` and add frontend TypeScript types so the plan page can display eligibility data.

# Files to Update

| File | Change |
|------|--------|
| `backend/app/modules/matching/engine.py` | Call `screen_benefits_eligibility()` in `generate_plan()` |
| `backend/app/modules/matching/types.py` | Add `benefits_eligibility` to `ReEntryPlan` |
| `frontend/src/lib/types.ts` | Add `ProgramEligibility`, `BenefitsEligibility`, `ProgramApplicationInfo` interfaces |

# Implementation Plan

1. **Update `ReEntryPlan`** in `matching/types.py`:
   - Add `benefits_eligibility: BenefitsEligibility | None = None`

2. **Update `generate_plan()`** in `engine.py`:
   - When `benefits_profile` present, call `screen_benefits_eligibility(benefits_profile)`
   - Attach to `plan.benefits_eligibility`
   - Runs alongside existing `calculate_cliff_analysis()` call

3. **Frontend types** in `types.ts`:
   - `ProgramApplicationInfo` interface
   - `ProgramEligibility` interface (program, eligible, confidence, income_threshold, income_headroom, estimated_monthly_value, reason, application_info)
   - `BenefitsEligibility` interface (eligible_programs, ineligible_programs, total_estimated_monthly, disclaimer)
   - Add `benefits_eligibility?: BenefitsEligibility | null` to `ReEntryPlan`

4. **Tests**: Verify generate_plan attaches eligibility when profile provided, null when not
