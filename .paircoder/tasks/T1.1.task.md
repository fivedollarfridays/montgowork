# Objective

Create all Pydantic type definitions across 6 modules. These are the typed contracts that Vinny (backend) and Shawn (frontend) will implement against. Every field, every enum value, every validator is the spec.

# Files to Update

| File | Change |
|------|--------|
| `backend/app/modules/matching/types.py` | BarrierType, BarrierSeverity, EmploymentStatus, AvailableHours, ScheduleConstraints, AssessmentRequest, UserProfile, Resource, JobMatch, TransitConnection, BarrierCard, ReEntryPlan |
| `backend/app/modules/credit/types.py` | AccountSummary, CreditProfileRequest, CreditAssessmentResult, score_to_band() |
| `backend/app/modules/data/types.py` | Employer, TransitRoute, TransitStop, WorkforceResource |
| `backend/app/modules/documents/types.py` | DocumentData |
| `backend/app/integrations/brightdata/types.py` | CrawlStatus, CrawlRequest, CrawlProgress, CrawlResult |
| `backend/app/ai/types.py` | AnalysisResult, PlanNarrative |

# Implementation Plan

1. Write `matching/types.py` first (largest, most referenced)
2. Write `credit/types.py` (includes AccountSummary, score_to_band helper)
3. Write `data/types.py` (DB-facing models)
4. Write `documents/types.py` (smallest)
5. Write `brightdata/types.py` (integration types)
6. Write `ai/types.py` (Claude API types)
7. Verify all imports work: `python -c "from app.modules.matching.types import ReEntryPlan"`
