# Objective

Create `backend/app/modules/benefits/eligibility_screener.py` that determines which Alabama benefit programs a household qualifies for based on income, household size, and dependents. This is the "opportunity system" — complementing Sprint 25's cliff "warning system" by telling users what programs they *can* get, not just what they might lose.

Follow the WIOA screener pattern: rule-based heuristic with confidence levels and a disclaimer.

# Implementation Plan

1. **Add types** to `benefits/types.py`:
   - `EligibilityConfidence(str, Enum)`: `likely`, `possible`, `unlikely`
   - `ProgramEligibility`: program, eligible, confidence, income_threshold, income_headroom, estimated_monthly_value, reason
   - `BenefitsEligibility`: eligible_programs list, ineligible_programs list, total_estimated_monthly, disclaimer

2. **Create `eligibility_screener.py`**:
   - `screen_benefits_eligibility(profile: BenefitsProfile) -> BenefitsEligibility`
   - Check ALL 7 programs against current income/household using thresholds from `thresholds.py`
   - Compute `income_headroom` = threshold - current_annual_income
   - Confidence: `likely` (well below threshold), `possible` (within 10%), `unlikely` (above)
   - Special cases: Medicaid adults (AL no expansion), ALL_Kids/Childcare require dependents, Section 8 waitlist, LIHEAP seasonal
   - Disclaimer: "This is an estimate. Contact DHR for official eligibility."

3. **Tests**: Each program, edge cases (zero income, max household, no dependents, threshold boundaries)
