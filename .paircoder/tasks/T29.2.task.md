# Objective

Add Alabama/Montgomery-specific application data for each benefit program — application URLs, required documents, office addresses, phone numbers, and processing times. Gives users actionable next steps: "You likely qualify for SNAP. Here's how to apply."

# Files to Update

| File | Change |
|------|--------|
| `backend/app/modules/benefits/application_data.py` | **NEW** — Per-program application steps, URLs, contacts |
| `backend/app/modules/benefits/types.py` | Add `ProgramApplicationInfo` model |
| `backend/app/modules/benefits/eligibility_screener.py` | Attach application info to eligible programs |
| `backend/tests/test_benefits_eligibility.py` | Add tests for application data |

# Implementation Plan

1. **Add `ProgramApplicationInfo`** to `types.py`:
   - application_url, application_steps, required_documents, office_name, office_address, office_phone, processing_time
   - Add field to `ProgramEligibility`

2. **Create `application_data.py`**:
   - `APPLICATION_DATA: dict[str, ProgramApplicationInfo]` with Montgomery-specific data:
     - **SNAP**: MyDHR portal, Montgomery County DHR, proof of income/ID/residency, 30 days
     - **TANF**: DHR in-person, work requirements docs, 30-45 days
     - **ALL_Kids**: Alabama Medicaid Agency, birth certificates, 45 days
     - **Childcare_Subsidy**: DHR childcare assistance, employment proof, provider selection
     - **Section_8**: Montgomery Housing Authority, waitlist application, 2-3 year wait
     - **LIHEAP**: DHR energy assistance, utility bill, seasonal (Oct-Mar)
     - **Medicaid**: Limited eligibility note, referral to navigator
   - `get_application_info(program: str) -> ProgramApplicationInfo | None`

3. **Wire into screener**: Attach application_info to each ProgramEligibility result

4. **Tests**: All programs have data, steps non-empty, screener includes app info
