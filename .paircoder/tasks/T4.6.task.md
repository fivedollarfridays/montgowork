# Objective

Build the jobs route from scratch (greenfield). Uses seed data only -- Bright Data integration is out of scope. Filter jobs by barrier compatibility, transit access, and skill alignment.

# Files to Update

| File | Change |
|------|--------|
| backend/app/routes/jobs.py | Add endpoints: GET /api/jobs, GET /api/jobs/{id} |
| backend/tests/test_jobs_route.py | NEW -- jobs route tests |

# Implementation Plan

1. `GET /api/jobs` -- list jobs with optional filters:
   - `?barriers=credit,transportation` -- filter by barrier compatibility
   - `?transit_accessible=true` -- only transit-accessible jobs
   - `?industry=healthcare` -- filter by industry
2. `GET /api/jobs/{id}` -- single job with application steps and resource connections
3. Use query layer to fetch from job_listings and employers tables
4. Apply transit filter: respect M-Transit schedule constraints (no Sunday, weekday ~5am-9pm)
5. Return with application steps and connected resources
6. Bright Data types exist but are NOT used -- seed data only
