# Objective

Implement focus management in the assessment wizard so keyboard users can navigate steps without losing context. When a step changes, focus moves to the new step's heading or first interactive element.

# Files to Update

| File | Change |
|------|--------|
| frontend/src/components/wizard/WizardShell.tsx | Add useRef for step content container, focus on step change |
| frontend/src/app/assess/page.tsx | Ensure step headings have tabIndex={-1} for programmatic focus |

# Implementation Plan

1. In WizardShell.tsx, add a `useRef<HTMLDivElement>` for the step content area.
2. Use `useEffect` watching `currentStep` — when it changes, call `stepRef.current?.focus()` or find the first heading within the step and focus it.
3. Add `tabIndex={-1}` to the step content container so it can receive programmatic focus without being in the tab order.
4. Ensure the "Previous" / "Next" / "Submit" buttons remain in logical tab order after step transitions.
5. Add `aria-current="step"` to the active step indicator in the step navigation.

# Acceptance Criteria

- [ ] When wizard advances to next step, focus moves to new step content
- [ ] When wizard goes back, focus moves to previous step content
- [ ] Step indicators show `aria-current="step"` on active step
- [ ] Tab order is logical: step content → form fields → navigation buttons
- [ ] No focus trap — user can Tab out of wizard to browser chrome
- [ ] Build passes
