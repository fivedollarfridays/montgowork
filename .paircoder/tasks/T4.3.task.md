# Objective

Implement the matching engine in `matching/engine.py`. This is the orchestrator that ties together queries, filters, and scoring to produce a `ReEntryPlan`.

# Files to Update

| File | Change |
|------|--------|
| backend/app/modules/matching/engine.py | Implement `generate_plan()` and `query_resources_for_barriers()` |
| backend/tests/test_engine.py | NEW -- engine orchestration tests |

# Implementation Plan

1. `query_resources_for_barriers(barriers, db_session)` -> list[Resource]:
   - Use query layer (T4.0) to fetch resources matching barrier types
   - Map BarrierType enum to resource categories
2. `generate_plan(profile, db_session)` -> ReEntryPlan:
   - Query resources via `query_resources_for_barriers()`
   - Apply filters: credit, transit, childcare (from T4.2)
   - Score and rank results (from T4.1)
   - Build BarrierCard objects with action steps per barrier
   - Populate job_matches, immediate_next_steps
   - Return complete ReEntryPlan
3. Add match_explanation via BarrierCard.actions for human-readable match reasons
4. Write tests with mocked DB session, filters, and scoring

# Acceptance Criteria
