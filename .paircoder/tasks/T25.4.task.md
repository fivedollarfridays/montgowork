# Objective

Add a benefits cliff chart to the plan page showing net income vs hourly wage. Cliff zones highlighted in red, matched jobs plotted as markers. Users instantly see which jobs are "safe" and which cause a benefits cliff.

# Implementation Plan

## 1. Backend — Cliff Data in Plan Response

Add `benefits_cliff_analysis: Optional[CliffAnalysis] = None` to `ReEntryPlan` model. In `engine.py`, if session has benefits_profile, run `calculate_cliff_analysis()` and attach result.

## 2. Frontend Types

- `WageStep`: wage, gross_monthly, benefits_total, net_monthly
- `CliffPoint`: hourly_wage, lost_program, monthly_loss, severity
- `CliffAnalysis`: wage_steps[], cliff_points[], current_net_monthly, worst_cliff_wage, safe_wage_floor

## 3. Chart Component (`BenefitsCliffChart.tsx`, ~100 lines)

Using Recharts:
- X-axis: Hourly wage ($8-$25, $0.50 steps)
- Y-axis: Monthly net income ($)
- Green line: total net income
- Red ReferenceArea: cliff zones (where net income drops)
- ReferenceDot: each matched job's wage level
- ReferenceLine: current income (dashed horizontal)
- Tooltip: "This job at $15/hr → net $2,100/mo (you currently net $2,300/mo)"
- ResponsiveContainer wrapping for mobile
- aria-label + text summary for screen readers

## 4. Plan Page Integration

New section after Job Matches, before Barrier Cards. Conditionally rendered only when `plan.benefits_cliff_analysis` exists. Includes text summary for accessibility.

## 5. Accessibility

- `role="img"` with descriptive aria-label
- Text summary: "Your biggest cliff is at $X/hr where you lose $Y/month in [program]"
- Color not sole indicator — cliff zones have labels
