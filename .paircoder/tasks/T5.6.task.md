# Objective

"What Changes in 3 Months" split-screen comparison. Left side: current situation (barriers active, limited jobs). Right side: projected situation after following the plan (barriers resolved, more jobs, higher earnings). Visual differentiator for judges.

# Files to Update

| File | Change |
|------|--------|
| frontend/src/components/plan/ComparisonView.tsx | Side-by-side Now vs Future comparison |

# Implementation Plan

1. Two-column layout (stacks on mobile): "Today" (left, muted) vs "In 3 Months" (right, vibrant)
2. "Today" column:
   - Barrier count with coral severity
   - Jobs eligible now (count)
   - Credit status (current score band)
   - Transit limitation note
3. "In 3 Months" column:
   - Barriers resolved (count with green checkmarks)
   - Jobs eligible after repair (larger count, green)
   - Projected credit improvement
   - Certifications renewed
4. Visual: left side uses muted/gray tones, right side uses green/teal success colors
5. Derive data from ReEntryPlan: eligible_now vs after_repair job counts, barrier list
