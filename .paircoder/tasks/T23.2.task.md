# Objective

Populate the `barrier_resources` join table by linking every existing resource in the DB
to the barrier nodes created in T23.1. Assign `impact_strength` scores (0.0–1.0) reflecting
how strongly each resource mitigates a barrier. Implement `get_top_resources_for_barriers()`
which the retrieval layer (T23.4) will call.

# Mapping Strategy

Map each resource `subcategory` → barrier node IDs:

```python
SUBCATEGORY_BARRIER_MAP = {
    "childcare": ["CHILDCARE_DAY", "CHILDCARE_EVENING"],
    "credit_counseling": ["CREDIT_LOW_SCORE", "CREDIT_NO_HISTORY"],
    "transportation": ["TRANSPORTATION_NO_CAR", "TRANSPORTATION_NO_TRANSIT"],
    "career_center": ["TRAINING_NO_CERT"],
    "social_service": ["HOUSING_UNSTABLE", "HEALTH_NO_INSURANCE"],
    "training": ["TRAINING_NO_CERT"],
}
```

Impact scores:
- Direct match: 0.9 | Strong match: 0.7 | Partial match: 0.5 | Weak match: 0.3

# Top-N Query Function

```python
async def get_top_resources_for_barriers(
    db: AsyncSession,
    barrier_ids: list[str],
    n: int = 5
) -> list[dict]:
    """Return top N resources ranked by combined impact_strength across given barriers."""
```

- JOIN `barrier_resources` → `resources`
- Filter by `barrier_id IN (barrier_ids)`
- Aggregate: `SUM(impact_strength)` per resource
- Exclude resources with `health_status = 'HIDDEN'`
- Return top N sorted by aggregate score descending
- Add SQL index: `CREATE INDEX idx_br_barrier ON barrier_resources(barrier_id)`
